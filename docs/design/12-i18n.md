# 12 — 本地化（i18n）

> 重写于 2026-06-10。Tk 时代的 i18n（`src/i18n.py` 承载全量 UI 字符串、重启切换、Phase 1-8 计划）已随 Tk 退役；旧版正文见 git 历史。

## 动机

VideoCraft 的第一波开源用户预计来自英文社区（Reddit、GitHub、Discord），但核心开发和早期用户是中文创作者。一套 UI 同时服务两类用户，通过 `tr(key)` 查表 + 用户可切换的语言偏好实现，无需维护两份代码。

## 现状

UI 主体在 Electron renderer，本地化也以 renderer 为主、Python 侧只剩少量：

| 层 | 文件 | 规模 | 说明 |
|----|------|------|------|
| renderer | [desktop/src/renderer/i18n/tr.ts](../../desktop/src/renderer/i18n/tr.ts) | — | `tr(key, vars?)` 同步查表 + `useLang()` 订阅；语言切换**热生效**（整树重渲染，无需重启） |
| renderer | [zh.json](../../desktop/src/renderer/i18n/zh.json) / [en.json](../../desktop/src/renderer/i18n/en.json) | 530 keys × 2 | 扁平表，zh/en 严格对称 |
| renderer | [LanguageToggle.tsx](../../desktop/src/renderer/i18n/LanguageToggle.tsx) | — | 语言切换控件；定制原生菜单（File/View/Help）标签同样随语言热切换 |
| Python | [src/i18n.py](../../src/i18n.py) + [src/i18n/](../../src/i18n/) | 28 keys × 2 | sidecar 侧少量面向用户的消息（`core/` 层不引 `tr`） |

## 机制

- **加载**：两张 locale 表静态 bundle 进 renderer，`tr()` 同步可用，任意组件直接调，无 provider。
- **种子**：boot 时 `main.tsx` await `system.get_locale`（sidecar 读 `user_data/settings.json` 的 `language` 键）→ `setLang()` → 首帧即正确语言。
- **切换**：`setLang` 通知订阅者 → `useLang` 触发重渲染 → 每个 `tr()` 重查新表；同时经 `system.set_locale` 持久化回 `settings.json`，跨会话存活。
- **fallback 链**（三级）：当前语言表 → en 表（factory default）→ 原样返回 key（防崩，同时是"未定义 key"的可见信号）。
- **插值**：`{name}` 占位符（与 Python `str.format` kwargs 对齐）；缺参时保留原占位文本，不崩。

## 纪律（不变）

1. **zh/en 双语同步**：改 i18n 必须两张表同改，key 集合严格对称。
2. **UI 新字符串必须走 `tr()`**，不许硬编码单一语言文案。
3. **UI 文案不准用代码内部名**：task name / field name / class name 不得直接出现在按钮、对话框、错误信息里。
4. **core 层本地化无关**：`src/core/` 不引 `tr`，消息由调用方组装。
