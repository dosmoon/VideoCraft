## ③ news_desk 剪裁 + crop-on-Clip 重构（`3894fd7` + 文档 `63a801d`，2026-06-08~09）

**需求**：给 news_desk 加「剪裁」。先分类清楚：剪裁 = **片段级空间变换**（不是叠加组件、不是全局属性；OTIO/NLE 一致放 Clip 层）。clip 早有此功能（reframe），但当时走渲染旁路 `DrawDeps.cropRect`，不在时间线上。

**用户拍板 3 条**：① 语义=空间裁剪/取景 ② 范围=news_desk 整条一个裁剪 ③ 架构=crop 落为 OTIO `Clip` 一等字段、退役 DrawDeps 旁路、clip/news_desk 统一。**权威 = [ADR-0011](../adr/0011-spatial-crop-clip-transform.md)**（foundation §2.5 加 `Clip.crop` + 不变量 #7）。

**8 阶段实现（每步 build-green）**：
- **S1** IR 加 `Clip.crop?: CropRect` + 校验不变量 #7（仅 media、`w,h>0`、⊆[0,1]）+ 单测。
- **S2+S3（原子）** compositor `draw.ts` 逐 media clip 读 `c.crop`、删 `DrawDeps.cropRect`；`encode.ts` 删 `ExportOptions.cropRect`；clip `buildClipTimeline` 设 crop、`ExportTab` 把 rect 路由进时间线而非 export deps。clip 行为逐字节不变（同 rect 同 shader，只换载体）；**预览从不 GPU 裁**（绿框是 2D 编辑提示）。
- **S4** news_desk config 加实例级 `outputMode/outputAspect/outputShortEdge/cropRect`，no-op `passthrough` 默认 = 整源直出不变。
- **S5** news_desk `assemble` 设 crop；`ExportTab` 按模式分支输出尺寸（passthrough=源 downscale / reframe+letterbox=目标比例 `targetDimsForAspect`）；`useNewsDeskPreview` 透出 framing。
- **S6** 几何提升到 `composition/crop.ts`（`CropRect` 与 IR 同源），改 6 个 importer，迁测试。
- **S7** 抽共享 `workbenches/shared/cropEditorLayer.ts`（`paintEditorLayer`/`canvasDimsFor`）；CropPreview 改用它；`NewsDeskPreview` 加拖框编辑器 + `StyleTab` 加 mode/aspect/short-edge 工具条；i18n +8 键（zh/en）。
- **S8** assemble 设/省 crop 测试。

**渲染语义**：reframe→设 crop（GPU cover）；letterbox→无 crop+fit contain（黑边）；passthrough→无 crop+输出尺寸=源。

**验证**：typecheck 干净 · vitest **232**（+13）· build OK · i18n **514×2 对称** · 零 Python 改动。用户真机验收通过。

**文档**（`63a801d`）：ADR-0011 + foundation §2.5（schema + 不变量 #7）+ ADR README。**重打包**（`a0b5d38` 戳记）：renderer-only（sidecar 未变），`VideoCraft-0.3.5-setup.exe` 2026-06-08 21:10 ~175MB，HEAD `63a801d`。
