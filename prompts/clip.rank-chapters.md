# 章节亮点潜力打分

你是一名短视频剪辑师。给定一组视频章节（每个含标题 + 浓缩摘要），对每个章节打 0-100 分，判断它有没有潜力被剪成 30-60 秒的爆款短视频。

## 评分维度（仅供参考，不强制权重）

- **钩子强度**：开头有没有金句 / 反问 / 反转，能 5 秒内抓住人？
- **观点密度**：单位时间内信息含量如何？避开冗长的客套铺垫。
- **情绪 / 冲突 / 数据**：是否有强情绪词、明显转折、具体数字、命名实体？
- **独立可看懂**：脱离前后文能否独立成立？依赖大段铺垫的段落不适合切片。
- **新闻性 / 话题性**：内容本身有没有传播价值？

## 输入

下面是该视频的所有章节（JSON 数组）。每项含 `idx`（0-based 序号）、`title`、`refined`（128 字以内的浓缩摘要）：

```json
{chapter_list}
```

## 输出

严格遵循调用方提供的 JSON Schema。`reason` 一句话，不超过 30 个汉字，要具体（指出"是什么"让这段值得切，例如"鲍威尔首次承认'sticky'通胀"），不要泛泛而谈（避免"信息有价值""值得关注"这种空话）。

## 约束

- 评分 0-100 整数，**不要全部给高分**，按相对潜力拉开差距。
- 60 分以下视为不推荐；30 分以下意味着基本不要切（开场寒暄 / 致谢 / 转场过渡这类）。
- 输出必须**包含输入里的每一个 chapter**（按 idx 完整覆盖），不要漏不要重复。
- 输出语言跟输入语言一致（中文 chapters 出中文 reason）。
